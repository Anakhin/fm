                Альтернативная история для менеджера FAR
                ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
                        Техническая информация
                        ~~~~~~~~~~~~~~~~~~~~~~

    В данном документе описаны настройки плагина, недоступные для управления
из интерфейса самого плагина.
    Все настройки хранятся в базе SQLite в файле 
    %FARPROFILE%\PluginsData\FF8FC1AE-0F35-4134-9BAB-56D71B1D47B9.db, 
    таблица table_values, если не указано иное


TechInfo #1
~~~~~~~~~~~
   "AepApplyNowDefault":SQLITE_INTEGER
   Параметр "AepApplyNowDefault" управляет состоянием флажка
   "Применить к существующей истории" при открытии окна
   "Добавить в список исключений".
   Может быть одним из следующих значений:
     0 - Флажок снят.
     1 - Флажок установлен.
   По умолчанию значение = 1 (Флажок установлен).


TechInfo #2
~~~~~~~~~~~
   "HiddenCommandsSpacesCount":SQLITE_INTEGER
   Параметр "HiddenCommandsSpacesCount" задает количество пробелов, которые
   должны предшествовать команде, попадающей в стандартную историю FAR,
   чтобы она не попала в историю команд AltHistory.
   Минимальное значение = 1, максимальное = 16.
   По умолчанию значение = 4.


TechInfo #4
~~~~~~~~~~~
   "MatchSelectedColor":SQLITE_INTEGER
   "MatchRegularColor":SQLITE_INTEGER
   Параметры "MatchSelectedColor" и "MatchRegularColor" определяют цветовой 
   аттрибут символов (кроме символа горячей клавиши) в элементе истории, 
   удовлетворяющих текущему фильтру (случаи для выделенного элемента и 
   прочих, соответственно).
   Могут быть одним из следующих значений:
     -1 - светло-зеленый цвет символов на текущем цвете фона.
     0-255 - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = -1 (светло-зеленый цвет символов на текущем 
   цвете фона).


   "MatchSelectedHighlightColor":SQLITE_INTEGER
   "MatchRegularHighlightColor":SQLITE_INTEGER
   Параметры "MatchSelectedHighlightColor" и "MatchRegularHighlightColor" 
   определяют цветовой аттрибут символа горячей клавиши в элементе истории,
   если символ удовлетворяет текущему фильтру (случаи для выделенного 
   элемента и прочих, соответственно).
   Могут быть одним из следующих значений:
     -1 - цвет символа не меняется (его определяет FAR), цвет фона становится 
          равным цвету фона, указанному в "MatchSelectedColor" или 
          "MatchRegularColor" (в зависимости от типа элемента)
     0-255 - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = -1.


TechInfo #5
~~~~~~~~~~~
   "CmdsHistoryLimitPolicy":SQLITE_INTEGER
   "ViewHistoryLimitPolicy":SQLITE_INTEGER
   "FoldersHistoryLimitPolicy":SQLITE_INTEGER
   Эти параметры управляют возможностью проводить автоматическое обновление
   соответствующих историй при записи их на диск.
   Могут принимать одно из следующих значений:
     0 - Автоматическое обновление не производится.
     1 - Производится автоматическое обновление аналогичное команде F5.
   Для "CmdsHistoryLimitPolicy" доступно еще одно значение:
     2 - Производится автоматическое обновление аналогичное команде Shift-F5.

   По умолчанию значение = 0 (автоматическое обновление не производится).

   Примечание: обновление проходит со следующими отличиями от оригинальных
   команд:
     1. Обновление происходит только если размер истории больше допустимого
        (указанного в диалоге конфигурации).
     2. Обновление прерывается, если размер обновляемой истории становится
        равным допустимому.
     3. При обновлении рассматриваются только пути, расположенные на локальных
        жестких дисках.


TechInfo #6
~~~~~~~~~~~
   "FilterColor":SQLITE_INTEGER
   Параметр "FilterColor" определяет цветовой аттрибут символов фильтра в 
   заголовке истории.
   Может быть одним из следующих значений:
     -1 - светло-зеленый цвет символов на цвете фона заголовка меню из 
     настроек FAR.
     0-255 - восьмибитное значение, младшие 4 бита кодируют цвет символов,
          старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = -1 (светло-зеленый цвет символов на цвете фона 
   заголовка меню из настроек FAR).


   "FilterWrongPartColor":SQLITE_INTEGER
   Параметр "FilterWrongPartColor" определяет цветовой аттрибут символов той 
   части фильтра, при которой было отфильтровано 0 элементов.
   Может быть одним из следующих значений:
     -1    - цвет выделеного текста меню из настроек FAR.
     0-255 - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = -1 (цвет выделеного текста меню из настроек FAR).


   "FilterAdditionColor":SQLITE_INTEGER
   Параметр "FilterAdditionColor" определяет цветовой аттрибут символов, 
   которые могут быть дописаны к фильтру без изменения отфильтрованных 
   элементов.
   Может быть одним из следующих значений:
     -1    - цвет недоступного текста меню из настроек FAR.
     0-255 - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = -1 (цвет недоступного текста меню из настроек FAR).

          настроек FAR.
     0-255 - восьмибитное значение, младшие 4 бита кодируют цвет
          символов, старшие 4 бита - цвет фона, по следующей схеме:
            76543210
            IRGBIRGB,
          где I - интенсивность цвета, RGB - соответственно - красная,
          зеленая и синяя компонента цвета.

   По умолчанию значение = -1 (светло-синий цвет символов на цвете фона 
   заголовка меню из настроек FAR).


TechInfo #9
~~~~~~~~~~~
   "BackSpaceRemoveFilterWrongPart":SQLITE_INTEGER
   Параметр "BackSpaceRemoveFilterWrongPart" управляет поведением плагина при
   нажатии BackSpace в окне истории.
   Может быть одним из следующих значений:
     0 - BackSpace всегда удаляет последний символ.
     1 - BackSpace удаляет неверную часть фильтра, если она присутствует,
         иначе удаляет последний символ.
   По умолчанию значение = 1 (BackSpace удаляет неверную часть фильтра, если
   она присутствует, иначе удаляет последний символ).
